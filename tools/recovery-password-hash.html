Récupération : remettre un password_hash dans Supabase

Contexte : si la colonne est vide ou si tu as effacé le hash, la connexion « pseudo + mot de passe » ne marche plus, même avec le bon mot de passe. Il faut insérer un nouveau hash (équivalent à celui que l’app enregistre).

  1. Saisis le même mot de passe que tu utilisais (ou un nouveau si tu veux le changer).
  2. Clique sur « Générer le hash ».
  3. Copie le bloc SQL en bas, remplace TON_PSEUDO si besoin.
  4. Dans Supabase → SQL → colle → Run.

Si la colonne password_hash n’existe plus, ré-exécute d’abord supabase/add_password_hash_to_profiles.sql.
